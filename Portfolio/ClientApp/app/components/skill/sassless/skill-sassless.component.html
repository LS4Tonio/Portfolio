Les générateurs de CSS

Les deux langages SASS, pour Syntactically Awesome Stylesheets, et LESS sont des préprocesseurs CSS. C’est-à-dire que ces langages permettent de générer du code CSS qui sera interprétable par un navigateur Web.

Pourquoi générer du code CSS plutôt que de faire du CSS ?
La raison est simple, une règle CSS est statique et lorsque l’on veut faire du style pour plusieurs éléments HTML on se retrouve vite avec une redondance de code et d’informations. La maintenance d’un tel code devient alors compliqué.
Les préprocesseurs ajoutent des mécanismes de variables, d’imbrications et de fonctions réutilisables.

Par exemple, imaginons que nous volons changer la couleur d’arrière-plan partagée dans plusieurs éléments. Dans du code CSS il faudrait remplacer toutes les occurrences de la couleur dans tous les éléments en question.
Avec un préprocesseur, la couleur d’arrière-plan serait associée à une variable et les éléments utilisent cette variable pour définir leur arrière-plan. En un seul changement, tout le code CSS généré aura la nouvelle valeur de la couleur.

L’utilisation de ses technologies a été un gain de temps pour l’entreprise et un approfondissement dans mes connaissances du CSS. J'ai pu établir des conventions et des thèmes rapidement adaptables aux clients et projets.

Mon portfolio utilise le préprocesseur LESS pour gérer le style. Toutes les couleurs du thème ainsi que les animations sont définies en un endroit et réutilisées dans les différents composants du site web.
